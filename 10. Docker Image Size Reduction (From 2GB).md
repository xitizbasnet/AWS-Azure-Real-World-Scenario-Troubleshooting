# Docker Image Size Reduction (From 2GB)

> **Scenario:** A Docker image size is approximately **2GB**, causing slow deployments, longer image pulls, and increased storage usage. The goal is to reduce the image size while maintaining application functionality.

---

# Overview

Large Docker images negatively impact:

* 🚀 Deployment speed
* 🌐 Container startup time
* 💾 Registry storage costs
* 📦 Image transfer time
* 🔄 CI/CD pipeline performance

The primary causes of oversized images are:

* Including unnecessary build dependencies
* Using large base images
* Copying unnecessary files
* Keeping development tools in production images
* Creating inefficient Docker layers

This guide explains Docker optimization techniques applicable to both **AWS ECR** and **Azure ACR** environments.

> 📌 **Important**
>
> Docker image optimization techniques apply equally to Amazon ECR and Azure Container Registry because both services store standard OCI/Docker images.

---

# AWS Approach

## Docker Image Optimization Techniques

Apply the following techniques in order of expected impact:

| Priority | Technique                    | Description                                                                                               |
| -------- | ---------------------------- | --------------------------------------------------------------------------------------------------------- |
| **1**    | 🏗️ Multi-stage Builds       | Build applications using a full image, then copy only required artifacts into a smaller production image. |
| **2**    | 📦 Alpine / Slim Base Images | Use lightweight images such as `node:18-alpine` instead of larger images.                                 |
| **3**    | 🧱 Combine RUN Commands      | Each `RUN` instruction creates a Docker layer. Combine related commands to reduce unnecessary layers.     |
| **4**    | 🚫 `.dockerignore`           | Exclude unnecessary files such as `node_modules`, `.git`, tests, and documentation.                       |
| **5**    | 🧹 Remove Build Tools        | Use `--no-install-recommends` and clean package caches in the same `RUN` instruction.                     |
| **6**    | 📂 Use COPY Instead of ADD   | `ADD` can automatically unpack archives and download URLs, creating larger layers.                        |

# Example: Before Optimization

## Single-Stage Docker Build (~2GB)

```dockerfile id="d9n3qa"
FROM node:18

WORKDIR /app

COPY . .

RUN npm install

RUN npm run build

CMD ["node", "dist/index.js"]
```

### Problems

* Includes full Node.js image.
* Includes source files.
* Includes build dependencies.
* Includes unnecessary development files.
* Keeps all build layers.

---

# Example: After Optimization

## Multi-Stage Docker Build (~120MB)

```dockerfile id="x8q4mz"
FROM node:18 AS builder

WORKDIR /app

COPY package*.json ./

RUN npm ci --only=production

COPY . .

RUN npm run build


FROM node:18-alpine

WORKDIR /app

COPY --from=builder /app/dist ./dist

COPY --from=builder /app/node_modules ./node_modules

USER node

CMD ["node", "dist/index.js"]
```

# Docker Ignore File

Create a `.dockerignore` file to exclude unnecessary content from the build context.

```text id="n5r2pk"
node_modules
.git
*.md
tests/
coverage/
.env*
```

Benefits:

* Smaller build context
* Faster Docker builds
* Reduced image size
* Prevents accidental inclusion of sensitive files

# Analyze Docker Image Layers

## Docker History

Review image layers:

```bash id="m3y8rv"
docker history prod-api:latest
```

---

## Dive Tool

Analyze image contents and identify unnecessary files:

```bash id="w6t1cx"
docker run --rm \
-v /var/run/docker.sock:/var/run/docker.sock \
wagoodman/dive prod-api:latest
```

> 💡 **Tip**
>
> The `dive` tool visually shows Docker image layers and helps identify files consuming unnecessary space.

# Root Cause Analysis (RCA)

## Root Cause

> ❌ The image contains unnecessary build components, including:

* Compiler tools
* Development dependencies
* Source maps
* Test files
* `.git` history
* Build artifacts

---

## Common Docker Layer Mistake

> ❌ Creating separate `RUN` commands for package installation and cleanup.

Example:

```dockerfile id="k8z5pf"
RUN apt-get update
RUN apt-get install package-name
RUN apt-get clean
```

Problem:

* Each command creates a separate layer.
* Deleted files may remain inside previous layers.

---

## Resolution

Apply the following optimization strategy:

```text id="r4c9hy"
Multi-stage Build
        +
Alpine/Slim Base Image
        +
.dockerignore
        =
Smaller Production Image
```

Expected reduction:

```text id="s2v7nk"
2GB → 150-200MB
```

for many Node.js and Python applications.

# Azure Approach

Azure uses the same Docker optimization principles with Azure Container Registry (ACR).

## Azure Container Registry (ACR)

### ACR Tasks

Enable automated image builds using:

* Multi-stage Dockerfiles
* Automated build triggers
* CI/CD integration

---

## ACR Content Trust

Use image signing and verification to:

* Validate image integrity.
* Ensure trusted image deployment.
* Protect against unauthorized image modifications.
