# Jenkins File Upload via User Input and SSH to Server

## 📘 Overview

This document describes how to implement a Jenkins-based file upload workflow where:

* A user uploads a file through a Jenkins input parameter.
* Jenkins processes the uploaded file.
* Jenkins connects securely to an Amazon EC2 target server using SSH.
* The uploaded file is transferred and placed in:

```text
/home/userhome-directory/
```

This approach uses SSH key-based authentication and is suitable for automated deployment workflows.

---

# AWS Approach

## 🏗️ Architecture

The deployment architecture consists of a Jenkins server communicating with a target Amazon EC2 server through SSH.

```text
Jenkins Server (EC2 or On-Premises)
        |
        |
        | SSH (Port 22)
        | Private key stored in Jenkins Credentials
        v
Target EC2 Server (Same VPC or VPN Connected)
        |
        |
        | Places file at:
        | /home/userhome-directory/
        v
File Available on Target Server
```

# ⚙️ Jenkins Pipeline Implementation

## Jenkinsfile (Declarative Pipeline)

```groovy
pipeline {
    agent any

    parameters {

        // This creates a file upload field in Jenkins UI
        base64File(
            name: 'UPLOADED_FILE',
            description: 'Upload file to deploy to server'
        )

        string(
            name: 'TARGET_USER',
            defaultValue: 'ec2-user',
            description: 'Linux username on target server'
        )

        string(
            name: 'TARGET_HOST',
            defaultValue: '10.0.10.25',
            description: 'Target server IP (private)'
        )

        string(
            name: 'REMOTE_PATH',
            defaultValue: '/home/ec2-user/',
            description: 'Remote path to place file'
        )
    }


    stages {

        stage('Prepare File') {

            steps {

                script {

                    // Decode the base64 file uploaded by user
                    def decodedFile = base64Decode("${params.UPLOADED_FILE}")

                    // Write file to Jenkins workspace
                    writeFile file: 'uploaded-file', text: decodedFile

                    echo "File prepared in workspace: ${WORKSPACE}/uploaded-file"
                }
            }
        }


        stage('Transfer File via SSH') {

            steps {

                // Method 1: Using SSH Agent Plugin (Recommended)

                sshagent(credentials: ['prod-ec2-ssh-key']) {

                    sh '''
                    # Verify target server connectivity

                    ssh -o StrictHostKeyChecking=no \
                    ${TARGET_USER}@${TARGET_HOST} "echo SSH OK"


                    # Transfer file using SCP

                    scp -o StrictHostKeyChecking=no \
                    ${WORKSPACE}/uploaded-file \
                    ${TARGET_USER}@${TARGET_HOST}:${REMOTE_PATH}


                    # Verify file arrived

                    ssh -o StrictHostKeyChecking=no \
                    ${TARGET_USER}@${TARGET_HOST} \
                    "ls -la ${REMOTE_PATH}"


                    echo "File successfully placed at ${TARGET_HOST}:${REMOTE_PATH}"
                    '''
                }
            }
        }


        stage('Verify Placement') {

            steps {

                sshagent(credentials: ['prod-ec2-ssh-key']) {

                    sh '''
                    ssh -o StrictHostKeyChecking=no \
                    ${TARGET_USER}@${TARGET_HOST} \
                    "ls -lh ${REMOTE_PATH} && md5sum ${REMOTE_PATH}/uploaded-file"
                    '''
                }
            }
        }
    }


    post {

        success {

            echo "File successfully deployed to ${TARGET_HOST}:${REMOTE_PATH}"
        }


        failure {

            echo "File transfer FAILED. Check SSH connectivity and permissions."
        }


        always {

            // Clean up local copy

            deleteDir()
        }
    }
}
```

# 🛠️ Jenkins Setup Requirements

Complete the following configuration steps before running the pipeline.

| # | Requirement                                                                                                          |
| - | -------------------------------------------------------------------------------------------------------------------- |
| 1 | Install **SSH Agent Plugin** from `Manage Jenkins → Plugins`                                                         |
| 2 | Install **File Parameter Plugin** for the `base64File` parameter type                                                |
| 3 | Add SSH key credential from `Manage Jenkins → Credentials → Add → SSH Username with Private Key`                     |
| 4 | Credential ID must match the pipeline reference: `sshagent(credentials: ['prod-ec2-ssh-key'])`                       |
| 5 | Jenkins EC2 security group must allow outbound TCP port `22` traffic to the target EC2 server                        |
| 6 | Target EC2 security group must allow inbound TCP port `22` traffic from the Jenkins EC2 IP address or security group |

# 🚀 Alternative Transfer Method: Using rsync

For large files, `rsync` can provide faster and more efficient transfer compared to SCP.

## Jenkins Pipeline rsync Example

```groovy
// In Jenkins pipeline step - rsync via SSH

sshagent(credentials: ['prod-ec2-ssh-key']) {

    sh '''
    rsync -avz --progress \
    -e "ssh -o StrictHostKeyChecking=no" \
    ${WORKSPACE}/uploaded-file \
    ${TARGET_USER}@${TARGET_HOST}:${REMOTE_PATH}
    '''
}
```

# 🔍 Root Cause Analysis (RCA)

## Common Failure Scenarios and Fixes

| Issue                                        | Root Cause Analysis                                                                     | Resolution                                                |
| -------------------------------------------- | --------------------------------------------------------------------------------------- | --------------------------------------------------------- |
| SSH connection hangs during first connection | `StrictHostKeyChecking` waits for manual `yes` confirmation                             | Use `-o StrictHostKeyChecking=no` for automated pipelines |
| Incorrect file location                      | Jenkins workspace path differs from the location where the plugin stores uploaded files | Verify the uploaded file path inside Jenkins workspace    |
| SSH timeout                                  | Security groups block port 22 communication between Jenkins and target server           | Allow SSH traffic between Jenkins and target EC2          |
| SSH key rejected                             | Private key permissions are too open                                                    | Set private key permissions to `400`                      |

---

## 📌 Production Recommendation

For production environments:

* Use:

```bash
-o StrictHostKeyChecking=no
```

for automated SSH pipelines.

* Store and manage `known_hosts` entries through Jenkins global configuration instead of disabling host verification permanently.

# Azure Approach

## ☁️ Azure DevOps Pipeline Equivalent

Azure DevOps provides similar functionality using SSH service connections and the Copy Files Over SSH task.

## azure-pipelines.yml

```yaml
parameters:
- name: fileToUpload
  displayName: 'Upload file'
  type: string


stages:

- stage: TransferFile

  jobs:

  - job: SCP

    steps:

    - task: SSH@0

      inputs:

        sshEndpoint: 'prod-server-ssh'

        runOptions: 'commands'

        commands: 'ls /home/azureuser/'


    - task: CopyFilesOverSSH@0

      inputs:

        sshEndpoint: 'prod-server-ssh'

        sourceFolder: '$(Build.SourcesDirectory)'

        contents: '*.txt'

        targetFolder: '/home/azureuser/'
```

---

# Azure DevOps Configuration Requirements

* Azure DevOps uses the **Copy Files Over SSH** task with an SSH Service Connection for the target server.
* Store SSH private keys securely using Azure Key Vault.
* Reference secrets from Key Vault during pipeline execution.

---

# 💡 Best Practice Security Tip

> Never store SSH private keys as plain text in Jenkins or Azure DevOps.

Always use:

* Jenkins Credentials Manager
* Azure Key Vault integration
* Secure secret management solutions

to protect SSH authentication credentials.
