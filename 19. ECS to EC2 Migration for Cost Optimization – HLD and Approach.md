# ECS to EC2 Migration for Cost Optimization – HLD and Approach

> **Scenario:**
> Frontend and Backend applications are currently running on ECS Fargate with:
>
> * Frontend: 2 vCPU / 4GB RAM
> * Backend: 2 vCPU / 4GB RAM
> * Current monthly cost: ~$170/month
>
> Objective:
>
> * Migrate workloads to EC2.
> * Keep the same VPC and subnet architecture.
> * Reduce monthly infrastructure cost.
> * Maintain container portability and deployment consistency.

# Migration Steps (Zero Downtime)

---

# Step 1: Launch EC2 Instances

Launch:

```text
2 × t3.large EC2 instances
```

Configuration:

* Existing VPC.
* Existing private subnets.
* Amazon Linux 2023 or existing AMI.

User Data:

* Install Docker.
* Authenticate with ECR.
* Pull images.
* Start containers.

---

# Step 2: Install Docker and Start Applications

Example User Data:

```bash
#!/bin/bash

yum update -y

yum install docker -y

systemctl start docker

aws ecr get-login-password --region ap-south-1 | \
docker login \
--username AWS \
--password-stdin <ecr-uri>

docker run -d \
--restart=always \
-p 3000:3000 \
--env-file /etc/app/.env \
<ecr-uri>/frontend:latest
```

---

# Step 3: Create New Target Groups

Create frontend target group:

```bash
aws elbv2 create-target-group \
--name ec2-frontend-tg \
--protocol HTTP \
--port 3000 \
--vpc-id vpc-abc123
```

Register EC2 instances:

```bash
aws elbv2 register-targets \
--target-group-arn <arn> \
--targets \
Id=i-0frontend123 \
Id=i-0frontend456
```

---

# Step 4: Update ALB Listener Rules

Actions:

1. Add EC2 target groups.
2. Keep ECS target groups temporarily.
3. Shift traffic gradually.

Traffic migration:

```text
20%
 |
50%
 |
100%
```

Monitor:

* CloudWatch metrics.
* HTTP errors.
* Response latency.
* Application logs.

---

# Step 5: Validate and Remove ECS

Validation period:

```text
48 hours monitoring
```

If successful:

1. Scale ECS service to:

```text
0 tasks
```

2. Delete ECS service.
3. Keep ECS cluster available for rollback.

Rollback window:

```text
1 week
```

# Root Cause Analysis (RCA)

## Why ECS Fargate Costs More

Cause:

```text
Fargate charges per vCPU/hour
even when workload is idle.
```

---

## Why EC2 Reserved Is Cheaper

For predictable workloads:

* Infrastructure runs continuously.
* Capacity requirements are stable.
* Reserved pricing reduces hourly cost.

---

# When NOT to Migrate Back to EC2

Do not migrate if:

* Traffic is unpredictable.
* Application requires scale-to-zero.
* Workload is highly bursty.

Example:

```text
Spiky workload:

Fargate is better
```

---

# Recommended Solution

For stable workloads:

```text
EC2 Reserved Instance

+

Docker Containers on EC2
```

Benefits:

* Same container portability.
* Same Docker images.
* Lower cost.
* More infrastructure control.

# Operational Best Practice

> 💡 **Tip**
>
> Always configure automatic container restart after EC2 reboot.

Recommended:

* systemd service.
* `/etc/rc.local`.
* Docker restart policy.

Example:

```bash
--restart=always
```

Without this:

```text
EC2 reboot

↓

Container stopped

↓

Manual intervention required
```
